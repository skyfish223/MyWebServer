## 30 秒回忆：调用顺序

```text
socket → setsockopt(可选但建议) → 填 sockaddr_in → bind → listen
  → while(true) { accept → read → 拼 HTTP 响应 → write → close(client_fd) }
```

## 系统调用速查（核心）

按代码里**第一次出现的顺序**排列。

| 名字 | 一句话（够回忆） | 生活类比 | 查参数时搜 |
|------|------------------|----------|------------|
| **`socket`** | 创建一个「网络听筒」 | 买一部电话 | `man 2 socket` |
| **`setsockopt`** | 给 socket 设开关/选项 | 给电话开通「来电显示」类功能 | `man 2 setsockopt` |
| **`bind`** | 把 socket 和「IP + 端口」绑死 | 给电话分配 8080 号码 | `man 2 bind` |
| **`listen`** | 进入「等客人敲门」状态 | 营业牌挂上，开始等电话 | `man 2 listen` |
| **`accept`** | 阻塞等到一个客人连进来，返回**新 fd** | 接起一通电话 | `man 2 accept` |
| **`read`** | 从某个 fd **读**字节到缓冲区 | 听对方说话 | `man 2 read` |
| **`write`** | 向某个 fd **写**字节 | 你回话 | `man 2 write` |
| **`close`** | 关闭 fd，释放连接/资源 | 挂电话 | `man 2 close` |
| **`htons`** | 把本机端口号转成**网络字节序** | 寄信前按邮局规定写地址格式 | `man 3 htons` |

### 各函数在流程里站哪一步

| 步骤 | 函数 | 操作的 fd |
|------|------|-----------|
| 开店 | `socket` → `setsockopt` → `bind` → `listen` | `server_fd` |
| 每来一个客人 | `accept` | 入：`server_fd`；出：`client_fd` |
| 听请求 | `read` | `client_fd` |
| 回响应 | `write` | `client_fd` |
| 送走客人 | `close` | `client_fd` |
| 程序结束（Demo 里很少走到） | `close` | `server_fd` |

## 结构体与类型

| 名字 | 一句话 | 和 HTTP 报文的关系 |
|------|--------|-------------------|
| **`sockaddr_in`** | 存 **IPv4 地址 + 端口** 的「地址本」 | ❌ 不是请求/响应正文；用于 `bind` / `accept` |
| **`sockaddr`** | 通用地址类型；API 要求用它的指针，所以常把 `sockaddr_in*` **强转**过去 | 同上 |
| **`socklen_t`** | 表示地址结构体**长度**的类型；`accept` 要传 | 同上 |
| **`ssize_t`** | **带符号**的「大小」类型；`read`/`write` 返回值用它（能表示 -1 错误） | — |
| **`int`（当 fd 用）** | 文件描述符，本质是**整数编号** | `server_fd` 监听；`client_fd` 单次连接 |

### `sockaddr_in` 里常填的三个字段（够用）

| 字段 | 记住 | 典型值（本篇） |
|------|------|----------------|
| `sin_family` | 地址族：IPv4 | `AF_INET` |
| `sin_addr.s_addr` | 监听哪个 IP | `INADDR_ANY`（本机所有网卡） |
| `sin_port` | 端口号（要 `htons`） | `htons(8080)` |

## 宏与常量

| 名字 | 一句话 |
|------|--------|
| **`AF_INET`** | 用 IPv4 |
| **`SOCK_STREAM`** | 用 TCP（可靠字节流） |
| **`INADDR_ANY`** | 监听本机所有网卡的 IP |
| **`SO_REUSEADDR`** | 允许端口快速重用（防 `bind` 失败） |
| **`SOL_SOCKET`** | `setsockopt` 的「选项级别」：套在 socket 这一层 |

## C++ 本篇用到的

| 名字 / 写法 | 一句话 |
|-------------|--------|
| **`std::string`** | 拼 HTTP 响应、方便算长度 |
| **`.c_str()`** | 把 `string` 转成 C 指针，给 `write` 用 |
| **`.size()`** | 字符串**内容**有多少字节（`write` 第三个参数） |
| **`std::string(buffer, n)`** | 用缓冲区前 `n` 字节构造字符串（安全打印） |
| **`std::min(a, b)`** | 取较小值（打印请求时只显示前 200 字符） |
| **`sizeof(数组/结构体)`** | 编译期固定大小；用于 `bind`、`read` 缓冲区等 |
| **`(sockaddr*)&addr`** | 经典写法：把 `sockaddr_in` 地址交给要 `sockaddr*` 的 API |

## HTTP 响应：要记住的名词

本篇**不解析**请求，但要会**拼**一段固定响应。不是函数，但天天见：

| 名词 | 一句话 |
|------|--------|
| **状态行** | 如 `HTTP/1.1 200 OK`：协议 + 成功码 |
| **`Content-Type`** | 告诉浏览器正文类型（本篇：`text/plain`） |
| **`Content-Length`** | 正文**有多少字节**（只算空行后面的 body，不是整段 response） |
| **`Connection: close`** | 发完就关连接 |
| **`\r\n`** | HTTP 行尾（不是单独 `\n`） |
| **空行 `\r\n\r\n`** | 响应头结束、正文开始的分界 |

拼响应口诀：**头 + 空行 + body**；改 body 时同步 `Content-Length`（或用 `body.size()` 自动生成，见深度答疑问题 14）。

## 两个 fd 别混

| 变量 | 角色 | 类比 |
|------|------|------|
| **`server_fd`** | 监听 socket，只负责「等连接、accept」 | 餐厅门口/总机 |
| **`client_fd`** | 某一次客人连接，用来 `read` / `write` | 当前这桌客人 |

`accept` 每次返回一个新的 `client_fd`；处理完必须 `close(client_fd)`，否则资源泄漏。

## 易混一张表

| 容易混 | 正确记法 |
|--------|----------|
| `sizeof(string)` vs `string.size()` | `write` 发内容用 **`.size()`**；`sizeof` 量的是对象壳子，不是 HTTP 长度（见答疑 13） |
| `Content-Length` vs `write(..., response.size())` | 前者只算 **body**；后者发 **整段响应**（头+body） |
| `sockaddr_in` vs `buffer` | 前者是**地址**；后者才是**HTTP 请求报文** |
| `read` 的 `sizeof(buffer)-1` vs 返回值 `n` | 前者是**最多读多少**；`n` 是**实际读了多少** |
| `bind` 失败 | 多半是 **8080 被占用**；加 `SO_REUSEADDR`、查 `ss -tlnp \| grep 8080`（见 01 指南 FAQ） |
