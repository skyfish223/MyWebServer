## 30 秒回忆：调用顺序

```text
（第一、二步不变）socket → bind → listen
  → while(true) {
        accept → read → rawRequest
        → parseRequestLine(rawRequest, req)
        → handleRequest(req)
              → isPathSafe(req.path)
              → pathToFile(req.path)      // URL → www/...
              → readFile(filePath, body)  // 磁盘读进 string
              → getContentType(filePath)  // 按后缀选类型
              → buildHttpResponse(200, OK, contentType, body)
        → write → close(client_fd)
     }
```

**第三步多出来的核心：** 页面在 **`www/` 磁盘文件**里，不再把 HTML 写死在 C++ 字符串里。

## 本篇新增四个函数 + 改造 handleRequest

| 名字 | 一句话 | 输入 → 输出 | 生活类比 |
|------|--------|-------------|----------|
| **`pathToFile`** | URL 路径 → 磁盘相对路径 | `urlPath` → `string` 如 `www/about.html` | 根据菜名找仓库货架 |
| **`isPathSafe`** | 禁止 `..` 越界访问 | `urlPath` → `bool` | 保安拦非法闯入 |
| **`getContentType`** | 按**文件后缀**选 MIME | `filePath` → `string`（Content-Type） | 给包裹贴「食品/衣物」标签 |
| **`readFile`** | 把整个文件读进 `string` | `filePath` + `out` → `bool` | 从货架整箱搬货 |
| **`handleRequest`**（改造） | 串联：安全检查 → 读文件 → 拼响应 | `HttpRequest` → 完整 HTTP `string` | 店长指挥全流程 |

第二篇保留、继续用：**`parseRequestLine`**、**`buildHttpResponse`**。

### 调用关系（handleRequest 内部）

```text
handleRequest(req)
  ├─ method != GET        → buildHttpResponse(405, ...)
  ├─ !isPathSafe(path)    → buildHttpResponse(403, ...)
  ├─ readFile 失败        → buildHttpResponse(404, ...)
  └─ 成功
        filePath = pathToFile(req.path)
        readFile(filePath, body)
        contentType = getContentType(filePath)
        → buildHttpResponse(200, OK, contentType, body)
```

## 文件读写：`ifstream` 与 `assign`

| 名字 / 写法 | 一句话 |
|-------------|--------|
| **`ifstream`** | **i**nput **f**ile stream，从文件读 |
| **`ifstream ifs(path, ios::binary)`** | 二进制打开（图片/CSS 不损坏） |
| **`ifs` 判断 `if (!ifs)`** | 打不开 → 文件不存在或路径错 |
| **`out.assign(迭代器1, 迭代器2)`** | 把 **[开头, 结尾)** 全部字符拷进 `string out` |
| **`istreambuf_iterator<char>(ifs)`** | 文件**开头** |
| **`istreambuf_iterator<char>()`** | 文件**结束**标记 |

读文件口诀：**`ifstream` 打开 → `assign` 整文件进 body**。

## MIME / Content-Type 对照

`getContentType` 看 **`filePath` 末尾后缀**（用 `substr(size - N)`）：

| 后缀 | 先判断 `size >=` | Content-Type |
|------|------------------|--------------|
| `.html` | 5 | `text/html; charset=utf-8` |
| `.css` | 4 | `text/css; charset=utf-8` |
| `.js` | 3 | `application/javascript; charset=utf-8` |
| `.png` | 4 | `image/png` |
| `.jpg` | 4 | `image/jpeg` |
| `.jpeg` | 5 | `image/jpeg` |
| 其它 | — | `application/octet-stream` |

**为什么要 `size >= N`？** 路径要够长才能安全取末尾 N 个字符当后缀（见 [03-深度答疑](./03-第三步代码深度答疑.md) 问题 2）。

## URL 路径 vs 磁盘路径

| 概念 | 例子 | 谁产生 |
|------|------|--------|
| **URL 路径** | `/about.html` | 浏览器请求行 → `req.path` |
| **磁盘相对路径** | `www/about.html` | `pathToFile(req.path)` |
| **绝对路径** | `D:\...\MyWebServer\www\...` | 系统：当前工作目录 + 相对路径 |

**请求里不会带 `C:\` 或 `D:\`**，磁盘路径是服务器自己拼的。

```text
GET /about.html  →  req.path="/about.html"  →  pathToFile  →  "www/about.html"  →  readFile
```

## 易混一张表

| 容易混 | 正确记法 |
|--------|----------|
| `req.path` vs `filePath` | 前者是 **URL**；后者是 **`www/...` 磁盘路径** |
| `pathToFile` vs `readFile` | 前者**算路径**；后者**读内容** |
| `getContentType(filePath)` vs `getContentType(req.path)` | 应对 **磁盘路径**（带 `.html` 后缀），两者通常一致 |
| `urlPath.find("..")` | 检查的是 **URL**，不是磁盘路径 |
| `substr(size - 5)` | 取**末尾 5 字符**（如 `.html`），不是从第 5 个开始 |
| `size >= 5` 和 `== ".html"` | **先**够长，**再**比对后缀 |
| `assign` vs `=` | 读整文件常用 **`out.assign(迭代器, 迭代器)`** |
| 文本模式 vs **`ios::binary`** | 图片/CSS 用 **binary** |
| 改 HTML | 改 **`www/*.html`**，不必改路由 if（同目录下） |
| 404 两种原因 | 文件真不存在；或 **工作目录不对**（找不到 `www/`） |
| 一次访问多个请求 | 首页 + **自动请求 CSS**（`/css/style.css`）是正常的 |

|--------|----------|

| **400** | 读不到请求 / 请求行格式错 |
