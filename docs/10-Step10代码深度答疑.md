## 问题1：`MYSQL*` 是什么？是 C++ 关键词吗？

### 相关代码

```cpp
#include <mysql/mysql.h>

MYSQL* conn = mysql_init(nullptr);
mysql_real_connect(conn, host, user, pass, db, port, nullptr, 0);
```

### 解答

#### 一句话

**`MYSQL` 不是 C++ 关键字，而是 MySQL C 客户端库（libmysqlclient）里定义的一个结构体类型；`MYSQL*` 是指向该结构体的指针，代表「一条与 MySQL 服务器的连接」。**

---

#### 拆开看

| 写法 | 含义 |
|------|------|
| `MYSQL` | 类型名（`struct`），在 `mysql.h` 里声明，对用户不透明（内部字段你看不到） |
| `*` | 指针：不拷贝整个连接对象，只拿「地址」 |
| `MYSQL* conn` | 变量 `conn` 保存一条连接的句柄，类似「电话线的编号」 |

C++ **关键字**是 `int`、`class`、`return` 这类语言保留字。  
`MYSQL` 是**第三方头文件**带来的类型，和 `FILE*`（标准 C 文件指针）是一类东西。

---

#### 生活类比

```text
MYSQL* conn  ≈  一根已经拨通 MySQL 的「专线电话」
mysql_query  ≈  对着这根电话说一句话（发 SQL）
mysql_close    ≈  挂断电话
```

连接池里 `queue<MYSQL*> pool_` 就是：池子里放着 8 根「已拨通的电话」，谁要用就 `get()` 借一根，用完 `release()` 还回去。

---

#### 和 Step10 里其它 MySQL 类型对照

| 类型 | 是什么 | 何时出现 |
|------|--------|----------|
| `MYSQL*` | 连接句柄 | `mysql_init`、`mysql_real_connect`、`mysql_query` |
| `MYSQL_RES*` | 查询结果集（多张「表」的容器） | `mysql_store_result` 之后 |
| `MYSQL_ROW` | 结果集里的一行（字符指针数组） | `mysql_fetch_row` 返回 |

下面问题 7 会细讲 `MYSQL_RES` 和 `MYSQL_ROW`。

## 问题2：`SqlPool` + `ConnectionRAII` 逐行详解

### 相关代码

```cpp
class SqlPool {
public:
    static SqlPool& instance() { static SqlPool p; return p; }

    void init(const char* host, int port, const char* user,
              const char* pass, const char* db, size_t poolSize) {
        for (size_t i = 0; i < poolSize; ++i) {
            MYSQL* conn = create_connection(host, port, user, pass, db);
            if (!conn) throw runtime_error("MySQL 连接池初始化失败");
            pool_.push(conn);
        }
        LOG_INFO("MySQL 连接池就绪 size=" + to_string(poolSize));
    }

    MYSQL* get() {
        unique_lock<mutex> lock(mtx_);
        cv_.wait(lock, [this]() { return !pool_.empty(); });
        MYSQL* conn = pool_.front();
        pool_.pop();
        return conn;
    }

    void release(MYSQL* conn) {
        if (!conn) return;
        lock_guard<mutex> lock(mtx_);
        pool_.push(conn);
        cv_.notify_one();
    }

private:
    SqlPool() = default;

    MYSQL* create_connection(const char* host, int port,
                             const char* user, const char* pass, const char* db) {
        MYSQL* conn = mysql_init(nullptr);
        if (!conn) return nullptr;
        if (!mysql_real_connect(conn, host, user, pass, db, port, nullptr, 0)) {
            LOG_ERROR(string("mysql_real_connect: ") + mysql_error(conn));
            mysql_close(conn);
            return nullptr;
        }
        mysql_set_character_set(conn, "utf8mb4");
        return conn;
    }

    queue<MYSQL*> pool_;
    mutex mtx_;
    condition_variable cv_;
};

struct ConnectionRAII {
    MYSQL* conn;
    ConnectionRAII() : conn(SqlPool::instance().get()) {}
    ~ConnectionRAII() { SqlPool::instance().release(conn); }
};
```

### 解答

#### 整体设计（先建立地图）

```text
SqlPool（单例）
  ├─ init()     启动时建好 N 条连接，放进 pool_
  ├─ get()      业务线程借一条连接
  ├─ release()  业务线程还连接
  └─ create_connection()  真正拨号连 MySQL

ConnectionRAII
  ├─ 构造：自动 get()
  └─ 析构：自动 release()   ← 防止忘记还连接
```

---

#### `public` 部分

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `static SqlPool& instance()` | **单例**：全局只存在一个连接池。`static SqlPool p` 在第一次调用时创建，之后都返回同一个 `p`。 |
| 2 | `void init(...)` | **预热连接池**。服务器启动时调用一次。 |
| 3 | `const char* host` | 数据库地址，如 `"127.0.0.1"`。`const char*` = 只读 C 风格字符串。 |
| 4 | `int port` | 端口，如 `3306`。 |
| 5 | `const char* user` / `pass` | 登录 MySQL 的用户名、密码。 |
| 6 | `const char* db` | 库名，如 `"webserver"`。 |
| 7 | `size_t poolSize` | 池大小，如 `8` = 预先建 8 条连接。 |
| 8 | `for (size_t i = 0; i < poolSize; ++i)` | 循环 `poolSize` 次，每次建一条连接。 |
| 9 | `MYSQL* conn = create_connection(...)` | 调用私有函数真正连接 MySQL，成功返回指针，失败返回 `nullptr`。 |
| 10 | `if (!conn) throw runtime_error(...)` | 任一条连接失败就抛异常，`main` 里 `catch` 后退出。 |
| 11 | `pool_.push(conn)` | 把连接指针放进队列尾部。 |
| 12 | `LOG_INFO(...)` | 打日志：池已就绪。 |

---

#### `get()` —— 借连接

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `unique_lock<mutex> lock(mtx_)` | 加锁，防止多个线程同时 `pop`/`push` 把池弄乱。`unique_lock` 比 `lock_guard` 更适合配合 `wait`。 |
| 2 | `cv_.wait(lock, [this]() { return !pool_.empty(); })` | **条件变量等待**：若池子空了（全被借走），当前线程睡在这里；别人 `release` 时会 `notify_one` 唤醒。`[this](){...}` 是 lambda：「池非空才停止等待」。 |
| 3 | `MYSQL* conn = pool_.front()` | 取队头那条连接（不弹出）。 |
| 4 | `pool_.pop()` | 从队头移除，表示「已借出」。 |
| 5 | `return conn` | 把连接交给业务代码去 `mysql_query`。 |

---

#### `release()` —— 还连接

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `if (!conn) return` | 空指针直接返回，防御性编程。 |
| 2 | `lock_guard<mutex> lock(mtx_)` | 加锁。 |
| 3 | `pool_.push(conn)` | 连接放回队尾。 |
| 4 | `cv_.notify_one()` | 唤醒一个正在 `get()` 里等待的线程（若有）。 |

---

#### `private` 部分

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `SqlPool() = default` | 私有默认构造，配合单例：外部不能 `SqlPool p;`，只能 `instance()`。 |
| 2 | `mysql_init(nullptr)` | 分配并初始化一个 `MYSQL` 结构，**尚未联网**。 |
| 3 | `mysql_real_connect(conn, host, user, pass, db, port, nullptr, 0)` | **真正连接** MySQL。参数依次：连接句柄、主机、用户、密码、库名、端口、unix socket（`nullptr` 不用）、客户端标志 `0`。返回 `conn` 表示成功，`nullptr` 表示失败。 |
| 4 | `mysql_error(conn)` | 连接失败时取错误描述字符串，写日志。 |
| 5 | `mysql_close(conn)` | 连接失败时释放已分配的资源。 |
| 6 | `mysql_set_character_set(conn, "utf8mb4")` | 设置字符集，支持中文。 |
| 7 | `queue<MYSQL*> pool_` | 空闲连接队列。 |
| 8 | `mutex mtx_` | 保护 `pool_` 的互斥锁。 |
| 9 | `condition_variable cv_` | 池空时阻塞 `get()`，有连接归还时唤醒。 |

---

#### `ConnectionRAII` 逐行

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `struct ConnectionRAII` | 用 `struct` 简单包一层，不是类层次设计。 |
| 2 | `MYSQL* conn` | 保存借来的连接指针。 |
| 3 | `ConnectionRAII() : conn(SqlPool::instance().get())` | **构造时**自动从池里 `get()` 一条连接。 |
| 4 | `~ConnectionRAII() { SqlPool::instance().release(conn); }` | **析构时**自动 `release()`，即使中间 `return` 或抛异常也会执行。 |

**RAII** = Resource Acquisition Is Initialization（资源获取即初始化）：  
资源（连接）的生命周期和对象生命周期绑定。

典型用法：

```cpp
{
    ConnectionRAII raii;
    MYSQL* conn = raii.conn;
    mysql_query(conn, sql);
}   // 离开作用域，析构自动还连接
```

## 问题3：`buildRedirectResponse` 是干嘛的？Step9 为什么没有？

### 相关代码

```cpp
string buildRedirectResponse(const string& location) {
    return "HTTP/1.1 302 Found\r\nLocation: " + location + "\r\n"
           "Content-Length: 0\r\nConnection: close\r\n\r\n";
}
```

### 解答

#### 一句话

**生成 HTTP 302 重定向响应，告诉浏览器：「别停在这个 URL，请立刻去 `Location` 里的地址再发一次 GET。」**

---

#### Step9 为什么没有、Step10 为什么需要？

| 步骤 | 典型响应 | 场景 |
|------|----------|------|
| Step9 | `200 OK` + HTML/纯文本 body | GET 静态页、POST `/echo` 回显 body |
| Step10 | **`302 Found` + `Location: /welcome.html`** | 注册/登录**成功后跳转**；失败跳 `/login_error.html` |

Step9 的 POST `/echo` 是「结果直接打在同一个响应 body 里」。  
Step10 的注册/登录是 Web 常见模式：**处理完表单 → 跳转到另一个页面**，不在 POST 响应里画整页 HTML。

```text
浏览器 POST /register  （提交表单）
    ↓
服务器 302 Location: /welcome.html
    ↓
浏览器自动 GET /welcome.html  （显示欢迎页）
```

---

#### 响应逐行含义

假设 `location = "/welcome.html"`，拼出的字符串是：

```http
HTTP/1.1 302 Found\r\n
Location: /welcome.html\r\n
Content-Length: 0\r\n
Connection: close\r\n
\r\n
```

| 部分 | 含义 |
|------|------|
| `HTTP/1.1 302 Found` | 状态码 302 = 临时重定向 |
| `Location: /welcome.html` | 浏览器下一步要请求的 URL（相对路径，相对当前站点） |
| `Content-Length: 0` | 响应体长度为 0（没有 HTML 内容，只有头） |
| `Connection: close` | 发完就关连接（与 Step6～10 一致，未做 Keep-Alive） |
| 最后的 `\r\n\r\n` | 空行：HTTP 规定头结束 |

---

#### 和 `buildHttpResponse` 的区别

```cpp
// Step3 起就有：200 + body
buildHttpResponse(200, "OK", "text/html", "<html>...</html>");

// Step10 新增：302，无 body，靠 Location
buildRedirectResponse("/welcome.html");
```

## 问题4：`url_decode` 举例逐行走读

### 相关代码

```cpp
string url_decode(const string& s) {
    string out;
    for (size_t i = 0; i < s.size(); ++i) {
        if (s[i] == '+') out += ' ';
        else if (s[i] == '%' && i + 2 < s.size()) {
            int hi = s[i+1], lo = s[i+2];
            auto hex = [](int c) { return (c>='0'&&c<='9')?c-'0':(c>='A'&&c<='F')?c-'A'+10:(c>='a'&&c<='f')?c-'a'+10:-1; };
            int h = hex(hi), l = hex(lo);
            if (h >= 0 && l >= 0) { out += static_cast<char>((h<<4)|l); i += 2; }
            else out += s[i];
        } else out += s[i];
    }
    return out;
}
```

### 解答

#### 为什么需要它？

HTML 表单 `method="post"` 默认编码是 **`application/x-www-form-urlencoded`**：

- 空格常编成 `+`
- 特殊字符编成 `%XX`（十六进制 ASCII）

例如用户名 `a b` 在 body 里可能是 `username=a+b` 或 `username=a%20b`。  
`url_decode` 把编码还原成真实字符串。

---

#### 例子 A：`s = "hello+world"`

| 步骤 | `i` | `s[i]` | 操作 | `out` |
|------|-----|--------|------|-------|
| 初始化 | — | — | `out = ""` | `""` |
| 循环 | 0 | `h` | 普通字符 `else out += s[i]` | `"h"` |
| | 1 | `e` | 同上 | `"he"` |
| | … | … | … | `"hello"` |
| | 5 | `+` | `out += ' '` | `"hello "` |
| | 6～10 | `world` | 逐字追加 | `"hello world"` |
| 结束 | — | — | `return out` | **`"hello world"`** |

---

#### 例子 B：`s = "a%20b"`（`%20` = 空格）

| 步骤 | `i` | 操作 | 说明 |
|------|-----|------|------|
| 0 | `a` | `out += 'a'` | 普通字符 |
| 1 | `%` | 进入 `%` 分支 | `i+2 < size` 成立 |
| | | `hi='2', lo='0'` | `i+1`、`i+2` 两个十六进制字符 |
| | | `hex('2')=2, hex('0')=0` | lambda 转数字 |
| | | `(2<<4)|0 = 32` | 即 ASCII 空格 |
| | | `out += ' '`，`i += 2` | `i` 从 1 跳到 3 |
| 3 | `b` | `out += 'b'` | |
| 返回 | | | **`"a b"`** |

---

#### 逐行说明

| 行 | 说明 |
|----|------|
| `string out` | 解码结果 |
| `for (size_t i = 0; i < s.size(); ++i)` | 从左到右扫描每个字符 |
| `if (s[i] == '+') out += ' '` | URL 编码里 `+` 表示空格 |
| `else if (s[i] == '%' && i + 2 < s.size())` | 遇到 `%` 且后面至少还有 2 个字符，才尝试 `%XX` |
| `int hi = s[i+1], lo = s[i+2]` | 取两位十六进制 |
| `auto hex = [](int c) { ... }` | 小 lambda：把 `'0'~'9'`、`'A'~'F'`、`'a'~'f'` 转成 0～15，非法返回 -1 |
| `int h = hex(hi), l = hex(lo)` | 高 4 位、低 4 位 |
| `out += static_cast<char>((h<<4)|l)` | 拼成一个字节：`h` 左移 4 位 OR `l` |
| `i += 2` | `%XX` 三个字符都处理完了，`for` 还会 `++i`，相当于总共跳过 3 个 |
| `else out += s[i]` | 普通字符原样拷贝 |
| `return out` | 返回解码串 |

## 问题5：`parse_form_urlencoded` 一句一句说明

### 相关代码

```cpp
map<string, string> parse_form_urlencoded(const string& body) {
    map<string, string> m;
    size_t start = 0;
    while (start < body.size()) {
        size_t amp = body.find('&', start);
        string pair = body.substr(start, amp == string::npos ? string::npos : amp - start);
        size_t eq = pair.find('=');
        if (eq != string::npos)
            m[url_decode(pair.substr(0, eq))] = url_decode(pair.substr(eq + 1));
        if (amp == string::npos) break;
        start = amp + 1;
    }
    return m;
}
```

### 解答

#### 输入输出长什么样

**输入** `body`（POST 正文）：

```text
username=test&passwd=123456
```

**输出** `m`：

```text
m["username"] == "test"
m["passwd"]   == "123456"
```

> 注意：`passwd` 是**表单字段名**（HTML `name="passwd"`），不是数据库列名 `password`。

---

#### 用例子 `body = "username=test&passwd=123456"` 走一遍

**第 1 轮循环**

| 语句 | 执行结果 |
|------|----------|
| `map<string, string> m` | 空 map |
| `size_t start = 0` | 从位置 0 开始 |
| `start < body.size()` | `0 < 28`，进入循环 |
| `amp = body.find('&', 0)` | 找到 `&` 在 index **8** |
| `pair = body.substr(0, 8-0)` | `"username=test"` |
| `eq = pair.find('=')` | `=` 在 index **8**… 不对，pair 是 `username=test`，`=` 在 index **8**？ `username` 长度 8，所以 `eq = 8` |
| `pair.substr(0, 8)` | `"username"` |
| `pair.substr(9)` | `"test"` |
| `m[url_decode("username")] = url_decode("test")` | `m["username"] = "test"` |
| `amp != npos` | 继续 |
| `start = 8 + 1 = 9` | 下一对从 `passwd=...` 开始 |

**第 2 轮循环**

| 语句 | 执行结果 |
|------|----------|
| `amp = body.find('&', 9)` | 没找到，返回 `npos` |
| `pair = body.substr(9, npos)` | `"passwd=123456"`（从 9 到结尾） |
| `eq = pair.find('=')` | `5`（`passwd` 后） |
| `m["passwd"] = "123456"` | |
| `amp == npos` | **`break`** 退出循环 |
| `return m` | 返回 map |

---

#### 逐句说明

| 行 | 说明 |
|----|------|
| `map<string, string> parse_form_urlencoded(...)` | 返回「键值对」字典：字段名 → 字段值 |
| `map<string, string> m` | 存放结果 |
| `size_t start = 0` | 当前解析到 body 的哪个下标 |
| `while (start < body.size())` | 还没扫完就继续 |
| `size_t amp = body.find('&', start)` | 从 `start` 起找下一个 `&`，用来分隔 `key=value` 对 |
| `string pair = body.substr(start, ...)` | 切出一对，如 `username=test`；若没有 `&`，切到字符串末尾 |
| `size_t eq = pair.find('=')` | 在这一对里找 `=`，左边是 key，右边是 value |
| `if (eq != string::npos)` | 有 `=` 才解析；畸形片段跳过 |
| `m[url_decode(...)] = url_decode(...)` | key、value 都要 URL 解码，写入 map |
| `if (amp == string::npos) break` | 没有更多 `&`，这是最后一对，结束 |
| `start = amp + 1` | 下一对从 `&` 后面开始 |
| `return m` | 返回完整表单 |

---

#### 和 `handle_register` / `handle_login` 的衔接

```cpp
auto form = parse_form_urlencoded(req.body);
string user = form["username"];  // 取「username」字段
string pass = form["passwd"];    // 取「passwd」字段
```

若 key 不存在，`form["xxx"]` 会插入空字符串，所以后面用 `empty()` 判断。

## 问题6：`escape_sql` 一行一行解释

### 相关代码

```cpp
string escape_sql(const string& s) {
    string out;
    for (char c : s) {
        if (c == '\'') out += "''";
        else if (c == '\\') out += "\\\\";
        else out += c;
    }
    return out;
}
```

### 解答

#### 为什么需要？

Step10 用字符串拼接 SQL：

```cpp
"... VALUES('" + escape_sql(user) + "','" + escape_sql(pass) + "')"
```

若用户名是 `O'Brien`，不转义会变成：

```sql
VALUES('O'Brien','xxx')   -- SQL 语法错误，引号提前结束
```

恶意用户还可能输入 `'; DROP TABLE user; --` 造成 **SQL 注入**（学习项目用转义演示，生产应用**预编译语句**）。

---

#### 例子：`s = "a'b"` 

| 轮次 | `c` | 分支 | `out` 变化 |
|------|-----|------|------------|
| 1 | `a` | `else out += c` | `"a"` |
| 2 | `'` | `out += "''"` | `"a''"`（SQL 里两个单引号表示一个 literal 单引号） |
| 3 | `b` | `else out += c` | `"a''b"` |

拼进 SQL：

```sql
VALUES('a''b', '...')
```

MySQL 会把 `a''b` 理解为一个字符串 **`a'b`**。

---

#### 逐行说明

| 行 | 说明 |
|----|------|
| `string escape_sql(const string& s)` | 输入原始用户字符串，返回「可安全放进单引号里」的版本 |
| `string out` | 结果累加器 |
| `for (char c : s)` | 逐字符扫描（范围 for） |
| `if (c == '\'') out += "''"` | 单引号 → 两个单引号（SQL 标准转义） |
| `else if (c == '\\') out += "\\\\"` | 反斜杠 → 两个反斜杠（MySQL 在部分模式下的转义） |
| `else out += c` | 其它字符原样保留 |
| `return out` | 返回转义后的串 |

## 问题7：`handle_login` 逐行解释（含 `MYSQL_RES`、`MYSQL_ROW`）

### 相关代码

```cpp
string handle_login(const HttpRequest& req) {
    auto form = parse_form_urlencoded(req.body);
    string user = form["username"];
    string pass = form["passwd"];
    if (user.empty() || pass.empty())
        return buildRedirectResponse("/login_error.html");

    ConnectionRAII raii;
    MYSQL* conn = raii.conn;
    string sql = "SELECT password FROM user WHERE username='"
               + escape_sql(user) + "' LIMIT 1";
    if (mysql_query(conn, sql.c_str()) != 0) {
        LOG_ERROR(string("登录查询失败: ") + mysql_error(conn));
        return buildRedirectResponse("/login_error.html");
    }
    MYSQL_RES* res = mysql_store_result(conn);
    if (!res) return buildRedirectResponse("/login_error.html");

    MYSQL_ROW row = mysql_fetch_row(res);
    bool ok = row && row[0] && string(row[0]) == pass;
    mysql_free_result(res);

    if (ok) {
        LOG_INFO("登录成功 user=" + user);
        return buildRedirectResponse("/welcome.html");
    }
    LOG_WARN("登录失败 user=" + user);
    return buildRedirectResponse("/login_error.html");
}
```

> **说明：** 你 ques 里写的是 `SELECT passwd`，当前 [step10.cpp](../step10.cpp) 已改为 `SELECT password`，与数据库列名一致；`form["passwd"]` 仍是读 HTML 表单字段，不变。

### 解答

#### 整体流程图

```text
POST body → 解析表单 → 取 username / passwd
    ↓
空？ → login_error.html
    ↓
RAII 借连接 → 拼 SELECT SQL → mysql_query 执行
    ↓
查询失败？ → login_error.html
    ↓
mysql_store_result 拿结果集 → mysql_fetch_row 取一行
    ↓
密码比对 → 成功 welcome / 失败 login_error
    ↓
mysql_free_result 释放结果集 → RAII 析构还连接
```

---

#### 逐行说明

| 行 | 代码 | 说明 |
|----|------|------|
| 1 | `string handle_login(const HttpRequest& req)` | 登录业务入口；`req` 里已有完整 POST（含 `body`）。 |
| 2 | `auto form = parse_form_urlencoded(req.body)` | 把 `username=xx&passwd=yy` 解析成 map。 |
| 3 | `string user = form["username"]` | 取用户名字段。 |
| 4 | `string pass = form["passwd"]` | 取密码字段（对应 HTML `name="passwd"`）。 |
| 5～6 | `if (user.empty() \|\| pass.empty())` | 任一为空 → 直接跳错误页，**不访问数据库**。 |
| 7 | `return buildRedirectResponse("/login_error.html")` | 302 到登录失败页。 |
| 8 | `ConnectionRAII raii` | 构造：从连接池借一条 `MYSQL*`。 |
| 9 | `MYSQL* conn = raii.conn` | 本地变量方便后面调用 C API。 |
| 10～11 | `string sql = "SELECT password FROM user WHERE username='" + escape_sql(user) + "' LIMIT 1"` | 拼 SQL：按用户名查**一条**密码。`LIMIT 1` 防止多行。 |
| 12 | `mysql_query(conn, sql.c_str()) != 0` | 把 SQL 发给 MySQL 执行。返回 **0 表示成功**，非 0 表示错误。 |
| 13 | `mysql_error(conn)` | 取错误信息写日志（如列名错、表不存在）。 |
| 14 | `return buildRedirectResponse("/login_error.html")` | 查询执行失败。 |
| 15 | `MYSQL_RES* res = mysql_store_result(conn)` | 把** SELECT 的查询结果**取到内存里，类型是「结果集指针」。 |
| 16 | `if (!res) return ...` | `nullptr`：可能没结果或这不是 SELECT；登录场景当失败处理。 |
| 17 | `MYSQL_ROW row = mysql_fetch_row(res)` | 从结果集取**第一行**。 |
| 18 | `bool ok = row && row[0] && string(row[0]) == pass` | 见下表拆解。 |
| 19 | `mysql_free_result(res)` | **释放结果集内存**，必须调用，否则泄漏。 |
| 20～24 | `if (ok)` 分支 | 成功 → 日志 + 跳 `welcome.html`；否则日志 + `login_error.html`。 |
| 25 | 函数结束 | `raii` 析构 → `release(conn)` 还连接。 |

---

#### `bool ok = row && row[0] && string(row[0]) == pass` 拆解

| 子表达式 | 含义 |
|----------|------|
| `row` | `mysql_fetch_row` 是否取到了行；`nullptr` = 没这个用户 |
| `row[0]` | 第一列（`SELECT password` 的那一列）指针非空 |
| `string(row[0]) == pass` | 库里的明文密码和表单密码一致（学习项目未做哈希） |

三者都真，`ok == true`。

---

#### `MYSQL_RES*` 是什么？

| 项目 | 说明 |
|------|------|
| 类型 | 指向「查询结果集」的不透明结构体 |
| 谁创建 | `mysql_store_result(conn)` |
| 里面有什么 | 多张逻辑「表」：行数、列数、每格字符串 |
| 谁释放 | `mysql_free_result(res)` |
| 类比 | 一整页 Excel 查询结果 |

只对 **SELECT** 这类返回数据的语句有意义；`INSERT` 一般用 `mysql_affected_rows`。

---

#### `MYSQL_ROW` 是什么？

| 项目 | 说明 |
|------|------|
| 类型 | 本质是 `char**`（字符指针数组）的 typedef |
| 谁创建 | `mysql_fetch_row(res)` 每次取一行 |
| `row[0]` | 第 1 列字符串（本例是 `password`） |
| `row[1]` | 第 2 列（若 `SELECT` 多列） |
| `nullptr` | 没有更多行了 |
| 类比 | Excel 里的**一行**；`row[0]` 是第一个单元格 |

示例：库里有一行 `password = "123456"`：

```text
row[0]  →  指向 "123456" 的 char*
string(row[0])  →  "123456"
```

---

#### 三种失败路径对照

| 现象 | 可能原因 | 日志关键词 |
|------|----------|------------|
| 立即 `login_error` | 表单空字段 | 无 MySQL 日志 |
| `登录查询失败` | SQL 语法/列名/表错 | `LOG_ERROR` + `mysql_error` |
| `登录失败 user=xxx` | 用户不存在或密码错 | `LOG_WARN` |

#### 和 `handle_register` 的对比（帮助记忆）

| | 注册 | 登录 |
|--|------|------|
| SQL | `INSERT INTO user(username, password) VALUES(...)` | `SELECT password FROM user WHERE username='...'` |
| 结果处理 | 看 `mysql_query` 返回值即可 | 还要 `store_result` → `fetch_row` → 比对 |
| 成功跳转 | `/welcome.html` | `/welcome.html` |
| 失败跳转 | `/register_error.html` | `/login_error.html` |

## 附录：易混概念速查

| 名字 | 在哪 | 是什么 |
|------|------|--------|
| `passwd` | HTML `name="passwd"`、POST body、`form["passwd"]` | **表单字段名** |
