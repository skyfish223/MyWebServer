## 30 秒回忆：调用顺序

```text
（第一步不变）socket → bind → listen
  → while(true) {
        accept
        → read → string rawRequest(buffer, n)
        → parseRequestLine(rawRequest, req)   // 抠出 GET /path HTTP/1.1
        → handleRequest(req)                  // 按 path 路由，内部调 buildHttpResponse
        → write(client_fd, response...)
        → close(client_fd)
     }
```

**第二步多出来的核心：** 不再写死 `Hello World`，而是 **解析请求行 → 路由 → 自动拼响应**。

## 本篇三个自定义函数

| 名字 | 一句话 | 输入 → 输出 | 生活类比 |
|------|--------|-------------|----------|
| **`parseRequestLine`** | 从原始报文里取出**第一行**，拆成 method / path / version | `const string& raw` + `HttpRequest& req` → `bool` | 听客人第一句：「我要 GET /about」 |
| **`handleRequest`** | 根据 **path** 决定返回什么（路由） | `const HttpRequest& req` → `string`（完整响应） | 厨师按菜单做菜 |
| **`buildHttpResponse`** | 把 **body** 加上 HTTP 响应头，拼成可发送的报文 | 状态码 + 类型 + body → `string` | 装盒、贴 Content-Length 标签 |

### 调用关系

```text
main:
  parseRequestLine(rawRequest, req)  ──失败──→ buildHttpResponse(400, ...)
       │
       └──成功──→ handleRequest(req)
                      │
                      ├── buildHttpResponse(200, ..., body)
                      ├── buildHttpResponse(404, ...)
                      └── buildHttpResponse(405, ...)
```

## C++ 字符串与流（解析用）

### `string` 成员函数

| 名字 | 一句话 | 典型写法 |
|------|--------|----------|
| **`find`** | 找子串**第一次**出现的位置；找不到返回 `npos` | `raw.find("\r\n")` |
| **`substr`** | 切一段子串 | `raw.substr(0, lineEnd)` 取第一行 |
| **`size`** | 字节个数（算 `Content-Length`、打印长度） | `body.size()` |

### 常量 / 类型

| 名字 | 一句话 |
|------|--------|
| **`size_t`** | 无符号整数，表示长度、下标（`find` 返回值） |
| **`string::npos`** | `find` **没找到**时的标记（必须带 `string::`） |

### 流：`istringstream`

| 名字 | 一句话 |
|------|--------|
| **`istringstream`** | **i**nput **string** **stream**：把一行 `string` 当输入源 |
| **`iss`** | 常见变量名（不是关键字） |
| **`iss >> a >> b >> c`** | 按**空白**切词，依次读进变量（不是 `req.method` 开头） |

### 其它 C++ 写法

| 名字 / 写法 | 一句话 |
|-------------|--------|
| **`string rawRequest(buffer, n)`** | 用 `read` 读到的前 `n` 字节**定义并初始化** string |
| **`const string&`** | 只读引用，不拷贝大字符串（如 `parseRequestLine` 的 `raw`） |
| **`HttpRequest&`** | 可写引用，把解析结果**写回** `main` 里的 `req` |
| **`to_string(数字)`** | 把 `statusCode`、`body.size()` 转成字符串拼进响应头 |

## 结构体与类型

| 名字 | 一句话 | 和 HTTP 的关系 |
|------|--------|----------------|
| **`HttpRequest`** | 存解析出的 **method、path、version** | 来自**请求行**，不是整段报文 |
| **`HttpRequest::method`** | 如 `GET`、`POST` | 请求行第一个词 |
| **`HttpRequest::path`** | 如 `/`、`/about` | 请求行第二个词，**路由靠它** |
| **`HttpRequest::version`** | 如 `HTTP/1.1` | 请求行第三个词 |

第一步的 `sockaddr_in`、`server_fd`、`client_fd` 等不变，见 [01-速查表 · 结构体](./01-第一步函数速查表.md#结构体与类型)。

## HTTP 请求：要记住的名词

| 名词 | 一句话 | 本篇要不要解析 |
|------|--------|----------------|
| **请求行** | 第一行：`GET /about HTTP/1.1` | ✅ **要**（`parseRequestLine`） |
| **请求头** | `Host:`、`User-Agent:` 等 | ❌ 先跳过 |
| **请求体** | GET 通常为空 | ❌ 先跳过 |
| **`\r\n`** | 行尾；第一行结束靠它定位 | ✅ 用来 `find` |
| **`rawRequest`** | `read` 之后转成的完整请求字符串 | 解析的输入 |

请求行口诀：**方法 + 路径 + 版本**，中间用空格分开。

---

## HTTP 响应：要记住的名词

| 名词 | 一句话 |
|------|--------|
| **状态行** | `HTTP/1.1 200 OK`：协议 + **状态码** + 短语 |
| **`statusCode`** | 数字：`200` 成功、`404` 没有、`405` 方法不允许（**不是** GET/POST） |
| **`statusText`** | 状态码后的英文，如 `OK`、`Not Found` |
| **`Content-Type`** | 正文类型：`text/html` 或 `text/plain; charset=utf-8` |
| **`Content-Length`** | 正文**字节数**；本篇用 `body.size()` **自动算** |
| **空行 + body** | `buildHttpResponse` 里 `"\r\n" + body` |

拼响应口诀：**状态行 + 头 + 空行 + body**（由 `buildHttpResponse` 一次拼好）。

---

## 状态码 vs 请求方法

| 概念 | 在哪出现 | 例子 | 存在哪个变量 |
|------|----------|------|--------------|
| **请求方法** | 请求行 | `GET`、`POST` | `req.method` |
| **状态码** | 响应行 | `200`、`404`、`405` | `buildHttpResponse` 的第一个参数 |

```text
浏览器发来：  GET /hello HTTP/1.1     ← method = GET
你发回去：    HTTP/1.1 200 OK         ← statusCode = 200
```

## 易混一张表

| 容易混 | 正确记法 |
|--------|----------|
| `find` vs `substr` | `find` **找位置**；`substr` **切子串** |
| `string::npos` vs `-1` | 没找到用 **`== string::npos`**，不是 `-1` |
| 第一个 `\r\n` vs 第二行 | 取第二行要从 `firstLineEnd + 2` 再 `find`（见深度答疑 4） |
| `iss >>` vs `req.method =` | 从**流**解析用 `>>`；手动赋值不用 `iss` |
| `const string& raw` vs `HttpRequest& req` | 前者**只读**输入；后者**写回**解析结果 |
| `rawRequest` vs `buffer` | `buffer` 是 `read` 的 C 数组；`rawRequest` 是 C++ string，方便解析 |
| `req.path` vs `statusCode` | path 是 URL；statusCode 是响应里的 200/404 |
| `Content-Length` vs `response.size()` | 前者只算 **body**；`write` 发的是**整段** response |
| `buildHttpResponse` vs `parseRequestLine` | 前者**拼回复**；后者**读请求** |
