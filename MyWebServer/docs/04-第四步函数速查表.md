## 30 秒回忆：调用顺序

```text
socket → bind → listen
  → setNonBlocking(listen_fd)
  → epoll_create1(0)  →  epfd
  → addFd(epfd, listen_fd)
  → while(true) {
        nready = epoll_wait(epfd, events, MAX_EVENTS, -1)
        for i in 0..nready-1:
            fd = events[i].data.fd
            if fd == listen_fd:
                while accept 直到 EAGAIN
                    setNonBlocking(client_fd)
                    addFd(epfd, client_fd)
            else:
                handleClient(epfd, fd)
                  → read → parse → buildResponseFromRequest
                  → write → removeFd → close
     }
```

**第四步多出来的核心：** 用 **epoll** 单进程监视 **listen_fd + 多个 client_fd**，不再 `accept` 阻塞串一整条链。

## epoll 三件套 + 工具函数

### epoll 核心 API

| 名字 | 一句话 | 生活类比 |
|------|--------|----------|
| **`epoll_create1(flags)`** | 创建 epoll 实例，返回 **`epfd`** | 准备一本空的值班表 |
| **`epoll_ctl(epfd, op, fd, &ev)`** | 往监视名单 **增 / 改 / 删** fd | 在值班表上登记或划掉桌号 |
| **`epoll_wait(epfd, events, max, timeout)`** | 等就绪事件，写入 `events`，返回 **`nready`** | 看哪几桌举手了 |

### 本篇封装的工具函数

| 名字 | 一句话 | 对应 API |
|------|--------|----------|
| **`setNonBlocking(fd)`** | fd 设为非阻塞 | `fcntl` + `O_NONBLOCK` |
| **`addFd(epfd, fd)`** | 把 fd 加入 epoll，监视 `EPOLLIN` | `epoll_ctl(ADD, ...)` |
| **`removeFd(epfd, fd)`** | 从 epoll 移除 fd | `epoll_ctl(DEL, ...)` |
| **`handleClient(epfd, client_fd)`** | 单连接：read → 业务 → write → DEL → close | 组合 read/write + epoll |

### 第三篇保留的业务函数

| 名字 | 第四步叫法 | 作用 |
|------|-----------|------|
| `handleRequest` | **`buildResponseFromRequest`**（可不改名） | `HttpRequest` → HTTP 响应字符串 |
| `parseRequestLine` | 同名 | 解析请求行 |
| `pathToFile` / `readFile` / `getContentType` | 同名 | 静态文件 www/ |

## handleClient vs buildResponseFromRequest

| | `buildResponseFromRequest` | `handleClient` |
|---|--------------------------|----------------|
| **层次** | 业务（等同 Step3 `handleRequest`） | I/O + 连接生命周期 |
| **参数** | `HttpRequest` | `epfd` + `client_fd` |
| **做 read/write？** | ❌ | ✅ |
| **做 epoll DEL？** | ❌ | ✅ `removeFd` |
| **做静态文件？** | ✅ | ✅（内部调用上层） |

```text
handleClient
  ├─ read / parseRequestLine
  ├─ buildResponseFromRequest(req)   ← 厨房做菜
  ├─ write
  ├─ removeFd
  └─ close
```

改名原因见 [04 指南 2.4 节](./04-epoll高并发指南.md#24-为什么把-handlerequest-改成-buildresponsefromrequest) · [深度答疑 问题1](./04-第四步代码深度答疑.md#问题1handleclient-和-handlerequest-有什么区别)。

## 常量与结构体

| 名字 | 一句话 | 典型值 |
|------|--------|--------|
| **`MAX_EVENTS`** | `epoll_wait` 一次最多返回几个事件（`events` 数组容量） | `64` |
| **`PORT`** | 监听端口 | `8080` |
| **`WEB_ROOT`** | 静态文件根目录 | `"www"` |
| **`epfd`** | epoll 实例 fd（`epoll_create1` 返回值） | 整数，如 3 |
| **`listen_fd`** | 监听 socket（只负责 accept） | 整数 |
| **`client_fd`** | 单次客户端连接（read/write） | 每次 accept 新建 |
| **`epoll_event`** | 描述「哪个 fd + 什么事件」 | `events` / `ev` |
| **`nready`** | 本次 `epoll_wait` 就绪个数 | for 循环上界 |

### `epoll_event` 里常填的字段

| 字段 | 本篇赋值 | 含义 |
|------|---------|------|
| `events` | `EPOLLIN` | 关心可读（listen=可 accept，client=可 read） |
| `data.fd` | `fd` | 就绪时知道是**哪个 fd** |

## fcntl 与非阻塞

```cpp
int flags = fcntl(fd, F_GETFL, 0);
fcntl(fd, F_SETFL, flags | O_NONBLOCK);
```

| 名字 | 一句话 |
|------|--------|
| **`fcntl`** | **f**ile **cntl**（文件控制） |
| **`F_GETFL`** | 读取 fd 当前标志 |
| **`F_SETFL`** | 设置 fd 标志 |
| **`O_NONBLOCK`** | 非阻塞：read/accept 无数据时返回 `EAGAIN`，不卡死进程 |

**谁要设非阻塞？** `listen_fd` 和每个 `client_fd`（`accept` 后、`addFd` 前）。

## epoll_ctl 操作对照

```cpp
epoll_ctl(epfd, op, fd, event_ptr);
```

| `op` | 封装函数 | 何时用 |
|------|---------|--------|
| **`EPOLL_CTL_ADD`** | `addFd` | 启动时加 `listen_fd`；`accept` 后加 `client_fd` |
| **`EPOLL_CTL_DEL`** | `removeFd` | `handleClient` 末尾，**close 前** |
| **`EPOLL_CTL_MOD`** | （本篇未用） | 修改监视的事件类型 |

### epoll_create1 / epoll_wait 参数速记

| 函数 | 关键参数 | 本篇取值 |
|------|---------|---------|
| `epoll_create1(flags)` | `flags` | `0`（默认） |
| `epoll_wait(..., timeout)` | `timeout` | `-1`（一直等到有事件） |
| `epoll_wait(..., maxevents)` | `maxevents` | `MAX_EVENTS`（64） |

详见 [04-深度答疑 问题6、7](./04-第四步代码深度答疑.md)。

## main 里 epoll 循环在干什么

### 启动阶段（只一次）

```text
socket → bind → listen
  → setNonBlocking(listen_fd)
  → epfd = epoll_create1(0)
  → addFd(epfd, listen_fd)
```

### 事件循环

| `fd == listen_fd` | 其它 fd（client） |
|-------------------|-------------------|
| `while (accept)` 直到 `EAGAIN` | `handleClient(epfd, fd)` |
| 每个新 `client_fd`：`setNonBlocking` + `addFd` | read → 静态文件响应 → write |
| 打印「新连接 fd=…」 | `removeFd` + `close` |

### 与 Step3 main 的差异

| Step3 | Step4 |
|-------|-------|
| `while { accept 阻塞; read; ...; close }` | `while { epoll_wait; 分发 }` |
| 业务全在 main | 单连接逻辑进 **`handleClient`** |
| 无 epoll | 有 **`epfd`**、**`addFd`**、**`removeFd`** |

## 易混一张表

| 容易混 | 正确记法 |
|--------|----------|
| `epfd` vs `listen_fd` vs `client_fd` | **epfd**=监视器；**listen**=接新客；**client**=单次连接 |
| `epoll_create1` vs `epoll_ctl` vs `epoll_wait` | **创建** / **改名单** / **等事件** |
| `addFd` vs `removeFd` | accept 后 **ADD**；handleClient 结束 **DEL** |
| `EPOLLIN` on listen vs client | listen=可 **accept**；client=可 **read** |
| `nready` vs `MAX_EVENTS` | **nready**=本次就绪数；**MAX_EVENTS**=数组容量上限 |
| `events.data()` vs `events[i].data.fd` | 传给 wait 的是数组头；取第 i 个就绪 fd |
| `EAGAIN` / `EWOULDBLOCK` | 非阻塞 accept/read **暂时没数据**，accept 循环应 **break** |
| `handleClient` vs `buildResponseFromRequest` | 前者管 **socket+epoll**；后者管 **HTTP+www** |
| `buildResponseFromRequest` vs `handleRequest` | **同一层业务**，名字不同而已 |
| 只 `close` 不 `removeFd` | 可能 epoll 仍监视坏 fd，应先 **DEL 再 close** |
| Step4 vs Step3 速度 | 本地自测差别不大；epoll 优势在**大量并发连接** |

## 附录：本篇 fd 生命周期

```text
listen_fd:
  创建 → 非阻塞 → addFd → 全程留在 epoll → 程序结束 close

client_fd:
  accept → 非阻塞 → addFd → epoll_wait 就绪 → handleClient
    → removeFd → close（本篇短连接，一次请求即关）
```

## 附录：epoll 事件流（简图）

```text
                    epoll_create1
                          │
                    addFd(listen_fd)
                          │
              ┌───── epoll_wait (-1) ─────┐
              │                           │
        listen_fd 就绪              client_fd 就绪
              │                           │
        while accept                  handleClient
        addFd(新 client)              removeFd + close
              │                           │
              └───────────┬───────────────┘
                          │
                    回到 epoll_wait
```
