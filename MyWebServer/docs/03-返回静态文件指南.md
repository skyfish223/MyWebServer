## 第一部分：先搞懂"我们在升级什么"

### 1.1 第二篇 vs 第三篇：差在哪？

| 对比项 | 第二篇 | 第三篇（本篇） |
|--------|--------|---------------|
| 页面内容存在哪 | **写在 C++ 代码里**（长字符串） | **放在磁盘** `www/` 文件夹 |
| 改页面要不要重编译 | 要改 `.cpp` 再编译 | 只改 `.html` / `.css`，**重启服务器即可** |
| CSS、图片 | 很难优雅地塞进字符串 | 浏览器自动请求 `/css/style.css` 等 |
| 路由方式 | 多个 `if (path == "/about")` | **URL 路径 ≈ 文件路径**（统一规则） |
| 生活类比 | 菜单印在服务员脑子里 | 菜单是**仓库里的真实菜谱本** |

### 1.2 什么是「静态文件」？

**静态文件**：内容已经完整存在于磁盘上，服务器**原样读出、原样发送**，不在服务器里动态生成。

| 类型 | 常见后缀 | 浏览器用来干什么 |
|------|---------|-----------------|
| 网页 | `.html` | 显示页面 |
| 样式 | `.css` | 美化页面 |
| 脚本 | `.js` | 交互（本篇可选） |
| 图片 | `.png` `.jpg` | 显示图片 |

### 1.3 URL 路径如何对应到磁盘文件？

约定一个**网站根目录**（本篇叫 `www`）：

```text
浏览器请求 URL              服务器去读的文件
─────────────────────────────────────────────
http://127.0.0.1:8080/          →  www/index.html
http://127.0.0.1:8080/about.html →  www/about.html
http://127.0.0.1:8080/css/style.css → www/css/style.css
```

规则可以记成：

```text
磁盘路径 = www + URL路径
（特殊：URL 是 / 时，默认读 index.html）
```

### 1.4 为什么必须告诉浏览器 Content-Type？

同一段字节，不同解释方式不同：

| Content-Type | 浏览器行为 |
|--------------|-----------|
| `text/html` | 当网页渲染 |
| `text/css` | 当样式表 |
| `image/png` | 当图片 |

若把 CSS 当成 HTML 发，页面会乱。所以要根据**文件后缀**选类型（叫 **MIME 类型**）。

### 1.5 本篇程序的整体流程

```text
  （第一、二篇已有：socket → accept → read → parseRequestLine）
     │
     ▼
  【新增】URL path → 拼出磁盘文件路径（www/...）
     │
     ▼
  【新增】安全检查（禁止 .. 越界访问）
     │
     ▼
  【新增】ifstream 读取文件内容到 string
     │
     ▼
  【新增】按后缀选择 Content-Type
     │
     ▼
  buildHttpResponse(200, ..., body) → write → close
     │
     └────── 文件不存在 → 404
```

### 1.6 一次访问首页时，浏览器其实会发多次请求

访问 `http://127.0.0.1:8080/` 时，终端可能依次出现：

```text
GET / HTTP/1.1           → 返回 index.html
GET /css/style.css ...   → 返回 style.css（HTML 里 <link> 触发的）
```

这是**正常现象**，说明静态资源链路通了。

## 第二部分：需要哪些新"工具"，为什么要用

### 2.1 新增头文件

| 头文件 | 提供什么 | 为什么需要 |
|--------|---------|-----------|
| `<fstream>` | `ifstream` 文件输入流 | 从磁盘**读**文件内容 |
| `<fstream>` 里也可用 | `ios::binary` | 按**二进制**读，图片不会读坏 |

第二篇已有 `<string>`、`<sstream>` 等**全部保留**。

### 2.2 本篇新增函数（分工）

| 函数 | 职责 | 生活类比 |
|------|------|---------|
| `pathToFile(path)` | 把 URL `/about.html` 转成 `www/about.html` | 根据菜单编号去仓库货架找货 |
| `isPathSafe(path)` | 检查是否含 `..` 等危险片段 | 保安：不许溜进仓库外的地方 |
| `getContentType(文件路径)` | 按后缀返回 MIME 类型 | 给包裹贴标签：这是食品/衣物 |
| `readFile(路径, body)` | 把文件整个读进 `string` | 把货架上的货整箱搬出来 |
| `handleRequest(req)` | 串联上面步骤，返回完整 HTTP 响应 | 店长指挥全流程 |

第二篇的 `parseRequestLine`、`buildHttpResponse` **继续用**，不用重写。

### 2.3 核心类：`ifstream`

```cpp
ifstream ifs("www/index.html", ios::binary);
```

| 部分 | 含义 |
|------|------|
| `ifstream` | **i**nput **f**ile stream，从文件读 |
| `ios::binary` | 二进制模式（图片、任意文件都适用） |

读完后放进 `string`，再交给 `buildHttpResponse`。

### 2.4 安全：为什么要防 `..`？

恶意 URL 可能写成：

```text
GET /../../etc/passwd HTTP/1.1
```

若直接拼接 `www + path`，可能读到**网站目录以外**的系统文件。  
本篇用简单规则：**path 里一旦出现 `..` 就拒绝**（返回 403 或 404）。

## 第四部分：完整代码 + 逐行讲解

### 4.1 完整代码

将下面保存为 **`Step1to6/Step3.cpp`**（或在 `Step1to6/Step2.cpp` 基础上改）：

```cpp
#include <iostream>
#include <fstream>
#include <sstream>
#include <string>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

using namespace std;

// 网站根目录：所有静态文件都放在这个文件夹下
const string WEB_ROOT = "www";

struct HttpRequest {
    string method;
    string path;
    string version;
};

// ---------- 第二篇已有：解析请求行 ----------
bool parseRequestLine(const string& raw, HttpRequest& req) {
    size_t lineEnd = raw.find("\r\n");
    if (lineEnd == string::npos) lineEnd = raw.find('\n');
    if (lineEnd == string::npos) return false;

    string line = raw.substr(0, lineEnd);
    istringstream iss(line);
    if (!(iss >> req.method >> req.path >> req.version)) return false;
    return true;
}

// ---------- 第二篇已有：拼 HTTP 响应 ----------
string buildHttpResponse(int statusCode,
                         const string& statusText,
                         const string& contentType,
                         const string& body) {
    return "HTTP/1.1 " + to_string(statusCode) + " " + statusText + "\r\n"
           "Content-Type: " + contentType + "\r\n"
           "Content-Length: " + to_string(body.size()) + "\r\n"
           "Connection: close\r\n"
           "\r\n" + body;
}

// ---------- 第三篇新增：URL → 磁盘路径 ----------
string pathToFile(const string& urlPath) {
    if (urlPath == "/" || urlPath == "/index.html") {
        return WEB_ROOT + "/index.html";
    }
    return WEB_ROOT + urlPath;  // 如 /about.html → www/about.html
}

// ---------- 第三篇新增：路径安全检查 ----------
bool isPathSafe(const string& urlPath) {
    // 禁止 .. 防止访问 www 以外的系统文件
    if (urlPath.find("..") != string::npos) return false;
    return true;
}

// ---------- 第三篇新增：按后缀选 Content-Type ----------
string getContentType(const string& filePath) {
    if (filePath.size() >= 5 && filePath.substr(filePath.size() - 5) == ".html")
        return "text/html; charset=utf-8";
    if (filePath.size() >= 4 && filePath.substr(filePath.size() - 4) == ".css")
        return "text/css; charset=utf-8";
    if (filePath.size() >= 3 && filePath.substr(filePath.size() - 3) == ".js")
        return "application/javascript; charset=utf-8";
    if (filePath.size() >= 4 && filePath.substr(filePath.size() - 4) == ".png")
        return "image/png";
    if (filePath.size() >= 4 && filePath.substr(filePath.size() - 4) == ".jpg")
        return "image/jpeg";
    if (filePath.size() >= 5 && filePath.substr(filePath.size() - 5) == ".jpeg")
        return "image/jpeg";
    return "application/octet-stream";  // 未知类型：二进制流
}

// ---------- 第三篇新增：读文件 ----------
bool readFile(const string& filePath, string& out) {
    ifstream ifs(filePath, ios::binary);
    if (!ifs) {
        return false;  // 文件不存在或无法打开
    }
    out.assign((istreambuf_iterator<char>(ifs)),
               istreambuf_iterator<char>());
    return true;
}

// ---------- 第三篇：改造 handleRequest ----------
string handleRequest(const HttpRequest& req) {
    if (req.method != "GET") {
        return buildHttpResponse(405, "Method Not Allowed",
            "text/plain; charset=utf-8",
            "暂只支持 GET，收到: " + req.method);
    }

    if (!isPathSafe(req.path)) {
        return buildHttpResponse(403, "Forbidden",
            "text/plain; charset=utf-8",
            "403 - 非法路径");
    }

    string filePath = pathToFile(req.path);
    string body;
    if (!readFile(filePath, body)) {
        return buildHttpResponse(404, "Not Found",
            "text/plain; charset=utf-8",
            "404 - 文件不存在: " + req.path);
    }

    string contentType = getContentType(filePath);
    return buildHttpResponse(200, "OK", contentType, body);
}

int main() {
    int server_fd = socket(AF_INET, SOCK_STREAM, 0);
    if (server_fd < 0) {
        cerr << "socket 创建失败\n";
        return 1;
    }

    int opt = 1;
    setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_addr.s_addr = INADDR_ANY;
    addr.sin_port = htons(8080);

    if (bind(server_fd, (sockaddr*)&addr, sizeof(addr)) < 0) {
        cerr << "bind 失败\n";
        close(server_fd);
        return 1;
    }

    if (listen(server_fd, 10) < 0) {
        cerr << "listen 失败\n";
        close(server_fd);
        return 1;
    }

    cout << "静态文件服务器已启动：http://127.0.0.1:8080\n";
    cout << "网站根目录: " << WEB_ROOT << "/\n";

    while (true) {
        sockaddr_in client_addr{};
        socklen_t client_len = sizeof(client_addr);
        int client_fd = accept(server_fd, (sockaddr*)&client_addr, &client_len);
        if (client_fd < 0) {
            cerr << "accept 失败\n";
            continue;
        }

        cout << "收到新连接\n";
        char buffer[8192] = {0};
        ssize_t n = read(client_fd, buffer, sizeof(buffer) - 1);

        string response;
        if (n <= 0) {
            response = buildHttpResponse(400, "Bad Request",
                "text/plain; charset=utf-8", "无法读取请求");
        } else {
            string rawRequest(buffer, n);
            HttpRequest req;
            if (!parseRequestLine(rawRequest, req)) {
                response = buildHttpResponse(400, "Bad Request",
                    "text/plain; charset=utf-8", "请求行格式错误");
            } else {
                cout << "解析结果: " << req.method << " " << req.path << "\n";
                cout << "读取文件: " << pathToFile(req.path) << "\n";
                response = handleRequest(req);
            }
        }

        write(client_fd, response.c_str(), response.size());
        close(client_fd);
        cout << "已响应并关闭连接\n";
    }

    close(server_fd);
    return 0;
}
```

### 4.2 逐块讲解：第三篇新增部分

#### `WEB_ROOT` 和 `pathToFile`

```cpp
const string WEB_ROOT = "www";

string pathToFile(const string& urlPath) {
    if (urlPath == "/" || urlPath == "/index.html")
        return WEB_ROOT + "/index.html";
    return WEB_ROOT + urlPath;
}
```

| URL path | 得到的路径 |
|----------|-----------|
| `/` | `www/index.html` |
| `/about.html` | `www/about.html` |
| `/css/style.css` | `www/css/style.css` |

**生活类比：** URL 是客人点的菜名，前面统一加仓库前缀 `www/` 就知道去哪拿货。

---

#### `isPathSafe`

```cpp
if (urlPath.find("..") != string::npos) return false;
```

`..` 表示「上一级目录」。出现它就拒绝，避免 `www/../../etc/passwd` 这类路径。

---

#### `getContentType`

根据**文件名后缀**返回 MIME 字符串，供 `Content-Type` 响应头使用。  
未知后缀用 `application/octet-stream`（通用二进制）。

---

#### `readFile`

```cpp
ifstream ifs(filePath, ios::binary);
out.assign((istreambuf_iterator<char>(ifs)),
           istreambuf_iterator<char>());
```

| 写法 | 含义 |
|------|------|
| `ifstream` | 打开文件 |
| `ios::binary` | 不转换换行，适合图片 |
| `istreambuf_iterator` | 把文件**从头到尾**读进 `string` |

失败（文件不存在）返回 `false`，上层返回 404。

---

#### 改造后的 `handleRequest`

第二篇是一大串 `if (path == "/about")`；第三篇变成**统一流程**：

```text
安全检查 → 算路径 → 读文件 → 选类型 → 拼响应
```

以后新增页面：**只要在 www 里加文件**，不必再改 C++ 路由表（同目录下）。

## 常见问题排查

### Q1：404，终端显示 `读取文件: www/index.html` 但打不开

**原因：** 运行 `./server` 时**当前工作目录**不对（不在项目根目录）。

**办法：**

```bash
cd /path/to/MyWebServer   # 能看到 www 文件夹
./server
```

或 `ls www/index.html` 确认文件存在。

### Q2：页面没样式，纯 HTML

**原因：** CSS 请求 404；或 `Content-Type` 不是 `text/css`。

**检查：** 终端有没有 `GET /css/style.css`；`getContentType` 是否识别 `.css`。

### Q3：中文乱码

HTML 里要有 `<meta charset="utf-8">`；`getContentType` 对 `.html` 使用 `charset=utf-8`。

### Q4：图片显示不出来

确认 `readFile` 用了 `ios::binary`；`Content-Type` 为 `image/png` 或 `image/jpeg`。

### Q5：`favicon.ico` 404

浏览器会自动请求 `/favicon.ico`。可在 `www/` 放一个图标，或忽略（不影响学习）。

### Q6：想访问 `www/img/logo.png`

把图片放进 `www/img/logo.png`，浏览器访问 `http://127.0.0.1:8080/img/logo.png` 即可，**不用改 C++**。
